Voice

MxClaw supports 4 voice providers for both real-time voice conversations and text-to-speech. The voice system is built on the VoicePlugin interface with a centralized VoiceManager.

Voice Providers

ProviderCapabilitiesConnectionConfig Key
OpenAI Realtime Voice input, voice output, streaming WebSocket (wss://api.openai.com/v1/realtime) openaiRealtime
ElevenLabs Voice output (TTS), streaming REST API + streaming elevenlabs
Gemini Live Voice input, voice output, streaming WebSocket (generativelanguage.googleapis.com/ws) geminiLive
System TTS Voice output (platform-native) Platform exec (say/espeak/SAPI) systemTts

VoicePlugin Interface

interface VoicePlugin {
  manifest: PluginManifest;
  initialize: (config: Record<string, unknown>) => Promise<void>;
  startSession: () => Promise<{ sessionId: string }>;
  sendAudio: (sessionId: string, audio: Buffer) => Promise<void>;
  receiveAudio: (sessionId: string) => AsyncGenerator<Buffer>;
  stopSession: (sessionId: string) => Promise<void>;
}

VoiceManager API

class VoiceManager {
  register(plugin: VoicePlugin): void;
  initialize(config: VoiceConfig): Promise<void>;
  startVoiceSession(provider: string): Promise<string>;  // returns sessionId
  sendAudio(sessionId: string, audio: Buffer): Promise<void>;
  receiveAudio(sessionId: string): AsyncGenerator<Buffer>;
  stopSession(sessionId: string): Promise<void>;
  getActiveSessions(): string[];
}

Text-to-Speech Utility

The textToSpeech() function in @mxclaw/voice provides direct TTS without a full session:

async function textToSpeech(
  text: string,
  provider: "elevenlabs" | "openai" | "system-tts",
  config?: Record<string, unknown>
): Promise<Buffer>

Platform-native TTS uses:

  • macOS — say command
  • Linux — espeak
  • Windows — System.Speech via PowerShell

Push-to-Talk

Voice conversations use the WebSocket protocol:

Client → ServerServer → Client
voice:start — begin sessionvoice:token — session ID returned
voice:audio — send audio chunk (base64)voice:error — error message
voice:stop — end sessionAudio streamed via receiveAudio generator

Per-agent voice config can override the default:

"agents": {
  "default": {
    "voice": {
      "provider": "elevenlabs",
      "voiceId": "21m00Tcm4TlvDq8ikWAM",
      "wakeWord": "hey mxclaw"
    }
  }
}
