TypeScript Configuration

All packages extend tsconfig.base.json which uses:

  • Target: ES2022
  • Module: ESNext
  • Resolution: bundler
  • Strict: true
  • Composite: true (incremental builds)
  • verbatimModuleSyntax: true
  • noUncheckedIndexedAccess: true

packages/gateway/src/__tests__/Gateway pipeline, message routing, webhook verification

Run tests:

pnpm test                    # Run all tests
pnpm test -- --reporter=verbose  # Verbose output
pnpm test:watch              # Watch mode

Adding a New Package

  1. Create packages/<name>/ with:
    • package.json — name as @mxclaw/<name>
    • tsconfig.json — extends ../../tsconfig.base.json
    • manifest.json — for plugin packages (type, capabilities, main)
    • src/index.ts — main source file
  2. Add package to the workspace (auto-detected via pnpm-workspace.yaml: packages/*)
  3. Run pnpm install to link workspace dependencies
  4. If it's a plugin, add it to root package.json exports if it needs public API access
  5. Add it to the Dockerfile's COPY steps for production builds

CI/CD Pipeline

GitHub Actions workflows in .github/workflows/:

CI (ci.yml)

  • Triggered on push/PR to main branches
  • Runs on ubuntu-latest with Node 22 + pnpm 9
  • Steps: lint → build → test

Release (release.yml)

  • Triggered on tag push (v*)
  • Steps: lint → build → test → docker build → docker push → npm publish

Docker Build

The Dockerfile uses a 3-stage build:

  1. deps — Install all dependencies (including devDeps) from frozen lockfile
  2. builder — Build all packages via pnpm -r build
  3. runner — Minimal production image with non-root user, exposes port 18700, healthcheck on /health
docker build -t mxclaw .
docker run -p 18700:18700 mxclaw

Or with docker-compose (includes Redis + Control UI nginx):

docker compose up -d
