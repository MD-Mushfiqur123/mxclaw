API Reference

WebSocket Gateway (Port 18700)

Clients connect to ws://127.0.0.1:18700 and must authenticate within 10 seconds. Authentication uses device tokens (8-char hex device ID + 32-byte random token, separated by :). Tokens rotate on every connect.

HTTP API

GET /health — Health check
GET /status — Full gateway status (channels, providers, sessions, memory)

Config

GET /api/config — Get current config (redacted secrets)
PUT /api/config — Update config (triggers hot-reload)

Sessions

GET /api/sessions?agentId=default — List sessions
GET /api/session/transcript?agentId=default&sessionKey=... — Get transcript
POST /api/session/reset — Reset a session

Approvals

GET /api/approvals — List pending approvals
POST /api/approval/resolve — Approve/deny

Pairing & Devices

POST /api/pairing/generate — Generate pairing code
POST /api/pairing/validate — Validate pairing code
POST /api/devices/pair — Pair a device

Chat

POST /api/chat/send — Send a chat message (accepts MessageEnvelope)

Skills

GET /api/skills — List loaded skills
POST /api/skills/toggle — Enable/disable a skill

Models

GET /api/models — List all models in catalog

Tokens

POST /api/token/generate — Generate API token

Webhooks

POST /gateway/webhook/:platform — Inbound webhook endpoint (signature verified)

Device Pairing Protocol

  1. Channel receives message from unknown sender → generatePairingCode(channelId, senderId)
  2. 8-char hex code sent back to sender via channel
  3. Sender enters code in Control UI → POST /api/pairing/validate
  4. On success, POST /api/devices/pair creates a device session with a 32-byte token
  5. Token used for WebSocket auth: { type: "auth", token: "deviceId:deviceToken" }
  6. Token rotates automatically on each connect via rotateDeviceToken()
