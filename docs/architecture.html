Architecture

Package Structure

MxClaw is a true pnpm monorepo with isolated, composable packages under packages/ and native apps under apps/.

@mxclaw/core

Types, 462-line Zod config schema, utilities. Zero runtime deps (just Zod).

@mxclaw/gateway

HTTP + WebSocket server, message routing engine, session management, agent execution pipeline.

@mxclaw/plugin-system

Plugin loader, registry, manifest-based auto-discovery from packages/.

@mxclaw/storage

JSONL-based transcript storage (default) + SQLite adapter. Embedding search via cosine distance over JSON files.

@mxclaw/security

Pairing codes, allowlists, mention gating, device token rotation, sandbox execution.

@mxclaw/tools

Agent tools: bash, browser (CDP), canvas (A2UI), cron, session_spawn, image_gen, file_r/w, web_search, web_fetch.

@mxclaw/voice

Voice integration: OpenAI Realtime WebSocket, ElevenLabs streaming TTS, Gemini Live, system TTS.

@mxclaw/logging

Subsystem-tagged logging with OTel support.

@mxclaw/skills

SKILL.md loader with YAML frontmatter + trigger matching + prompt builder.

@mxclaw/cli

Commander.js CLI with 16-provider onboarding wizard, gateway runner, diagnostics.

channel-* (22)

Individual channel plugins: Discord, Telegram, Slack, WhatsApp, etc.

provider-* (17)

Universal OpenAI-compat adapter + custom Bedrock SigV4.

Plugin System

The plugin system uses a registry pattern with manifest-based auto-discovery:

  1. Scan — On startup, loadPlugins() scans packages/ for directories containing manifest.json.
  2. Discover — Also scans node_modules/@mxclaw/ for npm-installed plugins.
  3. Filter — Only activates channel plugins if at least one channel of that type is configured.
  4. Register — Plugins are registered into PluginRegistry (channels, providers, voices maps).
  5. Error isolation — Failed plugins are skipped and surfaced in /status.

PluginManifest

interface PluginManifest {
  name: string;
  version: string;
  type: "channel" | "provider" | "tool" | "voice" | "extension";
  description: string;
  author: string;
  main: string;
  capabilities: string[];
  dependencies?: Record<string, string>;
}

Config Loading & Hot-Reload

Config is loaded from ~/.mxclaw/mxclaw.json:

  • Zod validation — Each load parses via MxClawConfigSchema (462 lines).
  • Snapshot fallback — On parse failure, falls back to mxclaw.snapshot.json.
  • Hot-reload — fs.watch monitors the config file; 500ms debounce; channels auto-restart on change.
  • Secrets resolution — $secret:KEY_NAME references resolved from AES-256-GCM vault.
  • Default generation — If no config exists, generateDefaultConfig() creates one with a default agent.

Skill System

Skills are loaded from SKILL.md files in the workspace:

  • YAML frontmatter with name, description, triggers, and tool definitions
  • Trigger matching routes messages to the correct skill
  • Prompt builder injects skill context into system prompt
  • Toggleable via Control UI or /api/skills/toggle endpoint
