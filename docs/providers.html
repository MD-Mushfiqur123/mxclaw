Providers

MxClaw uses a universal OpenAI-compatible adapter pattern — one adapter covers 17+ providers. Any provider that follows the OpenAI chat completions API works out of the box.

Additionally, dedicated standalone provider packages exist for specialized integrations:

  • AWS Bedrock — Custom SigV4 signing implementation
  • Anthropic — Native Anthropic API client
  • OpenAI — Native OpenAI SDK client
  • Google Gemini — Google AI SDK integration
  • Azure — Azure OpenAI endpoint
  • Cloudflare — Cloudflare Workers AI
  • Cohere — Cohere API
  • Replicate — Replicate API

ProviderPlugin Interface

interface ProviderPlugin {
  manifest: PluginManifest;
  initialize: (config: Record<string, unknown>) => Promise<void>;
  complete: (request: LLMCompletionRequest) => Promise<LLMCompletionResponse>;
  completeStream: (request: LLMCompletionRequest) => AsyncGenerator<LLMCompletionChunk>;
  listModels: () => Promise<Array<{ id: string; name: string }>>;
  healthCheck: () => Promise<boolean>;
}

Provider Config

Configure a provider in your agent's model field:

{
  "provider": "openai",
  "model": "gpt-4o",
  "apiKey": "$secret:OPENAI_API_KEY",
  "preset": "openai",
  "baseUrl": "https://api.openai.com/v1",
  "temperature": 0.7,
  "maxTokens": 4096,
  "modelAliases": { "fast": "gpt-4o-mini" }
}

Fallback Chain

Agents can define a fallback chain — if the primary provider fails, MxClaw retries with exponential backoff (max 3 retries) across the chain:

"fallbackChain": [
  { "provider": "anthropic", "model": "claude-sonnet-4-20250514" },
  { "provider": "groq", "model": "llama-3.3-70b-versatile" }
]

Adding a New Provider

  1. Create packages/provider-<name>/ with package.json, tsconfig.json, manifest.json, src/index.ts.
  2. Implement the ProviderPlugin interface.
  3. If it's OpenAI-compatible, add a preset to provider-openai-compatible/src/index.ts (around line 27).
  4. Add model entries to gateway/src/model-catalog.ts.
  5. Add to the CLI onboarding wizard.
