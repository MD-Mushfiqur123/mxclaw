Security

Secrets Vault (AES-256-GCM)

MxClaw stores API keys and credentials encrypted at rest using AES-256-GCM. The vault is a JSON file (~/.mxclaw/.secrets.vault) encrypted with a key derived from the machine ID or a master password.

class SecretsManager {
  constructor(workspacePath: string, masterPassword?: string);
  load(): Promise<void>;             // Load vault from disk
  save(): Promise<void>;             // Save vault to disk (encrypted)
  set(key: string, value: string): Promise<void>;
  get(key: string): string | undefined;
  delete(key: string): Promise<boolean>;
  listKeys(): string[];
  has(key: string): boolean;
  resolve(value: string): string;    // Resolve $secret:KEY references
}

Vault format: iv:tag:ciphertext (hex-encoded). Use $secret:KEY_NAME in your config to reference vault-stored values:

{
  "channels": {
    "discord-1": {
      "credentials": { "token": "$secret:DISCORD_BOT_TOKEN" }
    }
  }
}

Rate Limiting

Sliding window rate limiting with two classes:

  • Per-IP — IPRateLimiter: 100 requests per 60s window for HTTP API; returns 429 + Retry-After header
  • Per-WebSocket — Sliding window: 20 messages per second per client; returns RATE_LIMITED error
  • Per-Channel — ChannelRateLimiter: Configurable per-channel limits

Approval Gating

Tools support three approval modes (defined in @mxclaw/security):

ModeDescription
yoloAuto-approve — no confirmation needed
owner-onlyOnly the owner (matched by ownerId) can use without approval
always-require-approvalEvery invocation requires manual approval via Control UI or WebSocket

Approval requests are managed by ApprovalManager and can be resolved via WebSocket (chat:approve) or HTTP API (POST /api/approval/resolve). Pending approvals are visible on the Control UI Approvals page.

Device Pairing

  • Pairing codes — 8-char hex codes (crypto.randomBytes(4)), 5-minute expiry
  • Device tokens — 32-byte random tokens (base64url), SHA-256 hashed in memory
  • Token rotation — Tokens rotate on every WebSocket connect
  • QR codes — JSON payload: { type: "mxclaw-pairing", host, port, token, timestamp }

Sandbox Execution

interface SandboxConfig {
  enabled: boolean;
  type: "docker" | "ssh";
  image?: string;   // Docker image (default: mxclaw-sandbox:latest)
  host?: string;
  port?: number;
  username?: string;
}
  • Docker sandbox — docker run --rm --network none --read-only for untrusted bash commands
  • SSH sandbox — Remote execution over SSH for isolated environments

Additional Security

  • Mention gating — In group chats, only respond when @mentioned
  • Allowlists — Per-channel sender allowlists; empty allowlist = pairing mode
  • WebSocket auth timeout — Must authenticate within 10 seconds or connection is closed
  • Sender ID enforcement — WebSocket clients cannot impersonate other users
  • API token auth — Bearer token verification via timing-safe comparison
  • Config secrets resolution — $secret:KEY refs resolved at load time, never in raw config
