Channels

Channels are the messaging platform adapters that connect MxClaw to the outside world. Each channel plugin implements the ChannelPlugin interface and handles platform-specific authentication, message sending/receiving, and webhook verification.

ChannelPlugin Interface

Every channel must implement this interface (defined in @mxclaw/core):

interface ChannelPlugin {
  manifest: PluginManifest;
  setupChannel: (config: ChannelConfig) => Promise<void>;
  startChannel: (
    config: ChannelConfig,
    onMessage: (env: MessageEnvelope) => Promise<void>
  ) => Promise<void>;
  stopChannel: (channelId: string) => Promise<void>;
  sendMessage: (channelId: string, reply: ReplyEnvelope) => Promise<void>;
  handleCommand?: (channelId: string, command: string, args: string[]) => Promise<void>;
  handleApproval?: (channelId: string, approval: ApprovalRequest) => Promise<void>;
  getStatus: (channelId: string) => Promise<ChannelStatus>;
}

Config Fields

interface ChannelConfig {
  id: string;                    // Unique channel instance ID
  type: string;                  // Channel type (e.g. "discord")
  enabled: boolean;              // Whether this channel is active
  credentials: Record<string, unknown>;  // Tokens, secrets, etc.
  options: Record<string, unknown>;      // Platform-specific options
  allowlist: string[];           // Allowed sender IDs (empty = pairing)
  mentionGating: boolean;        // Only respond when @mentioned
  pairingEnabled: boolean;       // Allow unknown senders to pair
}

MessageEnvelope Format

All channels normalize inbound messages into this format:

interface MessageEnvelope {
  id: string;
  channel: string;               // Channel instance ID
  channelType: string;           // e.g. "discord", "telegram"
  sender: { id, displayName, avatarUrl?, isBot? };
  conversationId: string;
  threadId?: string;
  content: MessageContent[];     // Array of content parts
  mentions: string[];
  isGroupMessage: boolean;
  isMentioned: boolean;
  timestamp: number;
  metadata: Record<string, unknown>;
}

Content types (discriminated union):

type MessageContent =
  | { type: "text"; text: string }
  | { type: "image"; url: string; alt?: string }
  | { type: "audio"; url: string; transcript?: string }
  | { type: "video"; url: string }
  | { type: "file"; url: string; name: string; mimeType?: string }
  | { type: "location"; lat: number; lng: number; label?: string }
  | { type: "sticker"; id: string; url?: string }
  | { type: "reaction"; emoji: string; messageId: string }
  | { type: "canvas"; json: Record<string, unknown> }

Adding a New Channel

  1. Create packages/channel-<name>/ with package.json, tsconfig.json, manifest.json, and src/index.ts.
  2. Implement the ChannelPlugin interface.
  3. Add manifest.json with type: "channel" and capabilities list.
  4. Add webhook verification in gateway/src/webhook-verify.ts if applicable.
  5. Add a CLI preset in the onboarding wizard.
  6. Add to model catalog if the channel has special model requirements.
